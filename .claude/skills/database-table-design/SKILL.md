# PostgreSQL 数据库表设计规范

## 描述

gin-enterprise-template 项目 PostgreSQL 数据库表设计规范，包含建表模式、索引策略和最佳实践。

## 适用场景

- PostgreSQL 数据库表设计
- SQL 查询优化
- Schema 迁移脚本编写

## 核心规范

### 1. 表设计规范

#### 1.1 双主键设计

每张表使用双主键：内部自增 ID + 业务 UUID

```sql
CREATE TABLE "public"."example" (
    "id" bigint NOT NULL DEFAULT nextval('example_id_seq'::regclass),
    "example_id" uuid NOT NULL DEFAULT gen_random_uuid(),
    -- 其他字段...
    CONSTRAINT "example_example_id_key" UNIQUE ("example_id")
);
```

**设计理由**：
- `id` (bigint)：内部关联、分页排序高效
- `{resource}_id` (uuid)：对外暴露的业务标识，防止 ID 枚举攻击

> **注意**：项目使用 `gen_random_uuid()` 在数据库层生成 UUID（v4）。如需更高性能，可考虑应用层生成 UUID v7。

#### 1.2 标准时间戳字段

```sql
"created_at" timestamptz(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
"updated_at" timestamptz(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
"deleted_at" timestamptz(6),  -- 软删除时间（NULL=未删除）
```

#### 1.3 外键约束命名

```sql
CONSTRAINT "fk_{table}_{column}" FOREIGN KEY ("{column}")
    REFERENCES "public"."{ref_table}" ("{ref_column}")
    ON DELETE CASCADE ON UPDATE NO ACTION
```

> **注意**：项目使用 `ON DELETE CASCADE`，子表记录会随父表物理删除而级联删除。

### 2. 索引设计规范

#### 2.1 部分索引（软删除优化）

业务查询索引使用部分索引，过滤已删除数据：

```sql
CREATE INDEX "idx_example_active" ON "public"."example" ("created_at" DESC)
WHERE deleted_at IS NULL;
```

> **注意**：并非所有索引都需要软删除过滤，例如外键关联索引、路径查询索引等。

#### 2.2 业务唯一索引

```sql
-- 同一项目下名称唯一
CREATE INDEX "idx_example_unique" ON "public"."example" ("project_id", "name")
WHERE deleted_at IS NULL;
-- 或使用约束
ALTER TABLE "public"."example"
ADD CONSTRAINT "uni_example_project_name" UNIQUE ("project_id", "name");
```

#### 2.3 索引覆盖扫描（INCLUDE）

使用 `INCLUDE` 避免回表查询：

```sql
CREATE INDEX "idx_example_covering" ON "public"."example" ("task_id", "status")
INCLUDE ("example_id", "created_at")
WHERE deleted_at IS NULL;
```

#### 2.4 JSONB 字段 GIN 索引

```sql
CREATE INDEX "idx_example_jsonb_field" ON "public"."example"
USING GIN ("jsonb_field")
WHERE deleted_at IS NULL;
```

### 3. 注释规范

#### 3.1 字段注释

```sql
COMMENT ON COLUMN "public"."example"."id" IS '内部主键ID（自增序列）';
COMMENT ON COLUMN "public"."example"."example_id" IS '示例业务唯一UUID';
COMMENT ON COLUMN "public"."example"."project_id" IS '关联项目UUID，引用project(example_id)';
COMMENT ON COLUMN "public"."example"."name" IS '名称';
COMMENT ON COLUMN "public"."example"."created_at" IS '创建时间';
COMMENT ON COLUMN "public"."example"."updated_at" IS '更新时间';
COMMENT ON COLUMN "public"."example"."deleted_at" IS '软删除时间（NULL=未删除）';
```

#### 3.2 表注释

```sql
COMMENT ON TABLE "public"."example" IS '示例表：存储XXX的基本信息和统计概览';
```

#### 3.3 索引注释（复杂索引）

```sql
COMMENT ON INDEX "public"."idx_example_unique" IS '业务唯一索引：同一项目下名称唯一（支持软删除）';
```

### 4. 枚举类型规范

使用 PostgreSQL 枚举类型定义有限取值集合：

```sql
-- 创建枚举类型
DROP TYPE IF EXISTS "public"."example_type";
CREATE TYPE "public"."example_type" AS ENUM ('active', 'inactive');
ALTER TYPE "public"."example_type" OWNER TO "postgres";
COMMENT ON TYPE "public"."example_type" IS '示例类型枚举：active=激活, inactive=停用';

-- 表中使用枚举类型
CREATE TABLE "public"."example" (
    "id" bigint NOT NULL DEFAULT nextval('example_id_seq'::regclass),
    "example_id" uuid NOT NULL DEFAULT gen_random_uuid(),
    "status" "public"."example_type" NOT NULL DEFAULT 'active',
    -- ...
);
```

## 主键生成策略

### 1. 手动 SEQUENCE（项目采用）

项目所有表使用手动创建 SEQUENCE 的方式管理主键：

```sql
-- 第一步：创建序列
CREATE SEQUENCE "public"."example_id_seq"
    INCREMENT 1 MINVALUE 1 MAXVALUE 9223372036854775807 START 1 CACHE 1;
ALTER SEQUENCE "public"."example_id_seq" OWNER TO "postgres";
COMMENT ON SEQUENCE "public"."example_id_seq" IS '示例表内部ID序列';

-- 第二步：创建表时引用序列
CREATE TABLE "public"."example" (
    "id" bigint NOT NULL DEFAULT nextval('example_id_seq'::regclass),
    -- ...
);

-- 第三步：设置序列所有者
ALTER SEQUENCE "public"."example_id_seq" OWNED BY "public"."example"."id";
SELECT setval('"public"."example_id_seq"', 1, false);
```

**优点**：
- 兼容性强（所有 PostgreSQL 版本支持）
- 可跨表共享序列（如需要）
- 迁移脚本清晰

### 2. IDENTITY 替代方案（可选）

如不需要跨表共享序列，可使用 IDENTITY 简化：

```sql
CREATE TABLE "public"."example" (
    "id" bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    -- ...
);
```

## 外键与级联删除策略

### 项目采用 ON DELETE CASCADE

项目所有外键使用 `ON DELETE CASCADE`，子表记录会随父表物理删除而自动级联删除：

```sql
-- 示例：用户配置表外键
ALTER TABLE "public"."user_config"
ADD CONSTRAINT "user_config_user_id_fkey"
    FOREIGN KEY ("user_id") REFERENCES "public"."user" ("user_id")
    ON DELETE CASCADE ON UPDATE NO ACTION;

-- 示例：菜单表自关联外键
ALTER TABLE "public"."menu"
ADD CONSTRAINT "menu_parent_id_fkey"
    FOREIGN KEY ("parent_id") REFERENCES "public"."menu" ("menu_id")
    ON DELETE CASCADE ON UPDATE NO ACTION;
```

**适用场景**：软删除架构系统中，级联删除由业务层控制（先软删除子表，再软删除父表）。如果执行物理删除，则 CASCADE 会自动处理。

## 建表脚本模板

### 标准模板（手动 SEQUENCE）

```sql
-- ============================================================================
-- {模块名}数据库表创建脚本（事务性）
-- 说明：使用手动 SEQUENCE 管理主键，兼容所有 PostgreSQL 版本
-- ============================================================================
BEGIN;

-- ============================================================================
-- 第一步：删除旧表（级联删除依赖对象）
-- ============================================================================
DROP TABLE IF EXISTS "public"."example" CASCADE;
DROP SEQUENCE IF EXISTS "public"."example_id_seq" CASCADE;

-- ============================================================================
-- 第二步：创建序列
-- ============================================================================
CREATE SEQUENCE "public"."example_id_seq"
    INCREMENT 1 MINVALUE 1 MAXVALUE 9223372036854775807 START 1 CACHE 1;
ALTER SEQUENCE "public"."example_id_seq" OWNER TO "postgres";
COMMENT ON SEQUENCE "public"."example_id_seq" IS '示例表内部ID序列';

-- ============================================================================
-- 第三步：创建表（UUID 由数据库层 gen_random_uuid() 生成）
-- ============================================================================
CREATE TABLE "public"."example" (
    "id" bigint NOT NULL DEFAULT nextval('example_id_seq'::regclass),
    "example_id" uuid NOT NULL DEFAULT gen_random_uuid(),
    "project_id" uuid NOT NULL,
    "name" varchar(255) COLLATE "pg_catalog"."default" NOT NULL,
    "status" varchar(20) NOT NULL DEFAULT 'pending',
    "metadata" jsonb,
    "created_at" timestamptz(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updated_at" timestamptz(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "deleted_at" timestamptz(6),
    CONSTRAINT "example_example_id_key" UNIQUE ("example_id"),
    CONSTRAINT "fk_example_project_id" FOREIGN KEY ("project_id")
        REFERENCES "public"."project" ("example_id") ON DELETE CASCADE ON UPDATE NO ACTION
);
ALTER TABLE "public"."example" OWNER TO "postgres";

-- ============================================================================
-- 第四步：创建索引
-- ============================================================================
-- 业务唯一索引
CREATE UNIQUE INDEX "idx_example_unique" ON "public"."example" ("project_id", "name")
WHERE deleted_at IS NULL;

-- 常规查询索引
CREATE INDEX "idx_example_project_id" ON "public"."example" ("project_id");

-- 状态查询索引（软删除过滤）
CREATE INDEX "idx_example_status" ON "public"."example" ("status")
WHERE deleted_at IS NULL;

-- 排序优化索引
CREATE INDEX "idx_example_created_at" ON "public"."example" ("created_at" DESC)
WHERE deleted_at IS NULL;

-- ============================================================================
-- 第五步：添加注释
-- ============================================================================
COMMENT ON COLUMN "public"."example"."id" IS '内部主键ID（自增序列）';
COMMENT ON COLUMN "public"."example"."example_id" IS '示例业务唯一UUID';
COMMENT ON COLUMN "public"."example"."project_id" IS '关联项目UUID，引用project(example_id)';
COMMENT ON COLUMN "public"."example"."name" IS '名称';
COMMENT ON COLUMN "public"."example"."status" IS '状态：pending/processing/completed/failed';
COMMENT ON COLUMN "public"."example"."metadata" IS '扩展元数据（JSONB格式）';
COMMENT ON COLUMN "public"."example"."created_at" IS '创建时间';
COMMENT ON COLUMN "public"."example"."updated_at" IS '更新时间';
COMMENT ON COLUMN "public"."example"."deleted_at" IS '软删除时间（NULL=未删除）';
COMMENT ON TABLE "public"."example" IS '示例表：存储XXX的基本信息';

-- ============================================================================
-- 第六步：设置序列所有者
-- ============================================================================
ALTER SEQUENCE "public"."example_id_seq" OWNED BY "public"."example"."id";
SELECT setval('"public"."example_id_seq"', 1, false);

COMMIT;
```

## 索引设计最佳实践

### 1. 索引命名规范

| 索引类型 | 命名规则 | 示例 |
|---------|---------|------|
| 唯一约束 | `uni_{table}_{column}` 或 `{table}_{column}_key` | `uni_user_user_id`、`user_username_key` |
| 单列索引 | `idx_{table}_{column}` | `idx_menu_parent_id` |
| 组合索引 | `idx_{table}_{col1}_{col2}` | `idx_menu_parent_sort` |
| 覆盖索引 | `idx_{table}_{column}_covering` | `idx_task_status_covering` |
| GIN索引 | `idx_{table}_{column}_gin` | `idx_item_metadata_gin` |
| 部分索引 | `idx_{table}_active` | `idx_menu_active` |

### 2. 组合索引列顺序

```sql
-- ✅ 正确：等值查询列在前，范围/排序列在后
CREATE INDEX "idx_example_status_created" ON "public"."example" ("status", "created_at" DESC)
WHERE deleted_at IS NULL;

-- ❌ 错误：范围查询列在前，无法利用索引
CREATE INDEX "idx_example_created_status" ON "public"."example" ("created_at" DESC, "status")
WHERE deleted_at IS NULL;
```

### 3. 高频查询优化

```sql
-- 使用 INCLUDE 实现索引覆盖扫描
CREATE INDEX "idx_task_status" ON "public"."analysis_task" ("status", "created_at" DESC)
INCLUDE ("task_id", "name", "project_id")
WHERE deleted_at IS NULL;
```

### 4. 部分索引使用场景

| 场景 | 是否需要 WHERE 过滤 |
|------|---------------------|
| 软删除表的唯一性约束 | ✅ `WHERE deleted_at IS NULL` |
| 外键关联查询 | ❌ 直接查询子表 |
| 路径/编码查询 | ❌ 直接查询 |
| 时间范围统计 | ❌ 直接查询 |
| 状态过滤查询 | ✅ 大部分场景 |

## 常见陷阱与解决方案

### 1. N+1 查询问题

```sql
-- ❌ 错误：循环查询
-- SELECT * FROM items WHERE task_id = ? (N次)

-- ✅ 正确：使用 IN 或 JOIN
SELECT i.* FROM analysis_item i
WHERE i.task_id = ANY($1::uuid[]);
```

### 2. 缺少索引

```sql
-- 检查慢查询
EXPLAIN ANALYZE SELECT * FROM analysis_task WHERE status = 'pending';

-- 添加索引（使用 CONCURRENTLY 避免锁表）
CREATE INDEX CONCURRENTLY "idx_task_status" ON "public"."analysis_task" ("status")
WHERE deleted_at IS NULL;
```

### 3. 大事务问题

```sql
-- ❌ 错误：单次删除大量数据
DELETE FROM logs WHERE created_at < '2026-01-01';

-- ✅ 正确：分批删除
DELETE FROM logs WHERE created_at < '2026-01-01' LIMIT 1000;
-- 循环执行直到影响行数为0
```

### 4. UUID 索引性能

```sql
-- 项目使用数据库层 gen_random_uuid() 生成 v4 UUID
"example_id" uuid NOT NULL DEFAULT gen_random_uuid()

-- 优点：简单、保证唯一性
-- 注意：v4 完全随机，可能导致 B-tree 索引页分裂
-- 如需更高插入性能，可在应用层生成 UUID v7
```

## 数据维护建议

### 软删除数据清理

```sql
-- 定期清理90天前软删除的数据
DELETE FROM "public"."analysis_task" WHERE "deleted_at" < NOW() - INTERVAL '90 days';
```

### 索引维护

```sql
-- 分析表统计信息
ANALYZE "public"."analysis_task";

-- 重建碎片化索引
REINDEX INDEX CONCURRENTLY "public"."idx_task_status";
```

### 分区表维护

```sql
-- 查看分区情况
SELECT schemaname, tablename, partitionname
FROM pg_partitions
WHERE tablename = 'kafka_message_log';

-- 删除旧分区（比 DELETE 高效）
DROP TABLE IF EXISTS "public"."kafka_message_log_2026_01";
```
